Vēsture

Git pirmo reizi tika izlaists 2005.gadā. Taču 2007. gadā tika izveidots GitHub.


Git izstrāde sākās 2005. gada aprīlī, pēc tam, kad daudzi Linux kodola izstrādātāji atteicās no piekļuves BitKeeper (patentētai avotu kontroles pārvaldības (SCM) sistēmai, kuru viņi izmantoja projekta uzturēšanai kopš 2002.gada). Autortiesības BitKeeper īpašnieks Lerijs Makvojs bija atteicis produkta bezmaksas lietošanu pēc tam, kad apgalvoja, ka Endrjū Tridžels ir izveidojis SourcePuller, apgriežot BitKeeper protokolus. Tas pats incidents pamudināja izveidot citu versiju kontroles sistēmu Mercurial.


Linuss Torvalds vēlējās izplatītu sistēmu, ko viņš varētu izmantot tāpat kā BitKeeper, taču neviena no pieejamajām bezmaksas sistēmām neatbilda viņa vajadzībām. Torvalds minēja piemēru, kad avota kontroles pārvaldības sistēmai ir vajadzīgas 30 sekundes, lai lietotu ielāpu un atjauninātu visus saistītos metadatus, un atzīmēja, ka tas neatbilst Linux kodola izstrādes vajadzībām, kur sinhronizācijai ar citiem uzturētājiem varētu būt nepieciešamas 250 šādas darbības vienreiz. Savam dizaina kritērijam viņš norādīja, ka lāpīšanai vajadzētu aizņemt ne vairāk kā trīs sekundes, un pievienoja vēl trīs mērķus:

  1. Kā piemēru tam, ko nevajadzētu darīt, izmantojiet vienlaicīgu versiju sistēmu (CVS); šaubu gadījumā pieņemiet tieši pretēju lēmumu.
  2. Atbalstiet izplatītu, BitKeeper līdzīgu darbplūsmu.
  3. Iekļaujiet ļoti spēcīgus aizsardzības pasākumus pret nejaušu vai ļaunprātīgu korupciju.

Šie kritēriji likvidēja visas tajā laikā izmantotās versiju kontroles sistēmas, tāpēc tūlīt pēc 2.6.12-rc2 Linux kodola izstrādes laidiena Torvalds nolēma rakstīt savu.


Darbošanās gaita:

  • Torvalds par projektu paziņoja 6. aprīlī un nākamajā dienā kļuva par pašu saimniekošanu.
  • Pirmā vairāku filiāļu apvienošana notika 18. aprīlī. Torvalds sasniedza savus rezultativitātes mērķus;
  • 29. aprīlī topošajam Git tika veikts Linux kodola koka ierakstīšanas ielāpu salīdzinošais novērtējums ar ātrumu 6,7 ielāpi sekundē.
  • 16. jūnijā Git pārvaldīja kodola 2.6.12 versiju.

Torvalds 2005. gada 26. jūlijā nodeva apkopi Junio ​​Hamano, kas ir galvenais projekta ieguldītājs. Hamano bija atbildīgs par 1.0 laidienu 2005. gada 21. decembrī un joprojām ir projekta galvenais uzturētājs.


Autors


Linuss Torvalds (dzimis 1969. gada 28. decembrī, Helsinkos, Somijā), somu datorzinātnieks, kurš bija galvenais Linux operētājsistēmas izstrādes spēks. Viņš ir arī versiju kontroles sistēmas Git autors. 2012. gadā ieguva Tūkstošgades tehnoloģiju balvu.

10 gadu vecumā Torvalds sāka nodarboties ar datorprogrammēšanu sava vectēva Commodore VIC-20.



Mērķi un mērķauditorija

Mērķauditorija: Koledžas studenti, Git SCM iesācēji vai tie, kas vēlas pāriet no citiem SCM rīkiem uz git.

Tās mērķi ietver ātrumu, datu integritāti, atbalstu izplatītai, nelineārai darbplūsmu sadarbībai starp failu apdarinātājiem. Git izseko failos veiktās izmaiņas, lai jums būtu ieraksts par paveikto un, ja vēlaties, varat atgriezties pie konkrētām versijām. Git arī atvieglo sadarbību, ļaujot vairāku cilvēku veiktās izmaiņas apvienot vienā avotā. Reāllaika projektā var būt daudz līdzstrādnieku vai izstrādātāju, kas strādā, lai izstrādātu, un arī ir pārliecība, ka izmaiņas notiks jaunu funkciju, kļūdu dēļ, kuru rezultātā tiek veikti labojumi. Tāpēc koordinācijas nodrošināšanai mēs izmantojam versiju kontroles sistēmu, lai izvairītos no konfliktiem un arī uzturētu viņu darba vēsturi.
